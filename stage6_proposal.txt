### Архітектурні рішення

**Технологія:** PySide6 (Qt 6.8 LTS) + pyqtgraph (custom candlestick
renderer). Вже встановлено в Етапі 1.

**Async-Qt bridge:** `AsyncWorker` (QThread + asyncio loop) — вже
реалізовано в Етапі 2 через `MarketDataService`.

**Threading model:**
- Main thread: тільки UI (рендеринг, event loop Qt)
- AsyncWorker thread: BinanceClient, scoring engine
- Cross-thread signals: Qt автоматично робить QueuedConnection

### Scope Етапу 6 (MVP)

### Опції реалізації

**Варіант A: Монолітний Етап 6 (~1000 рядків)**

Один великий етап: MainWindow + ChartWidget + ScorePanel + integration.
Результат — готовий Radar одразу.

Плюси:
- Швидше отримуємо працюючий інструмент
- Менше coordination overhead між етапами
- Один повний "user journey" за один етап

Мінуси:
- Більше коду за раз = більше ризик помилок
- Довша ітерація без проміжних результатів
- Складніше review

**Варіант B: Розділений на 6A + 6B (~500 + 500 рядків)**

**6A: Каркас + Chart (1-2 дні)**
- MainWindow, SymbolPanel, ChartWidget з candlesticks
- Можна вибирати символи, бачити графіки з EMA
- Без scoring panel

**6B: ScorePanel + integration (1-2 дні)**
- ScorePanel з breakdown
- Підключення до MarketDataService
- Refresh button
- Повний "Radar" готовий

Плюси:
- Частіші проміжні результати
- Легше review кожної частини
- Можна "покатати" 6A без scoring, оцінити UI/UX

Мінуси:
- Дві ітерації замість однієї
- 6A без scoring panel виглядає "напівпоробленим"

**Варіант A (монолітний Етап 6)** — тому що:

## Технічні деталі (для реалізації)

### pyqtgraph candlesticks

pyqtgraph не має вбудованих candlesticks. Потрібно реалізувати:
1. Малювати high-low як вертикальну лінію
2. Малювати open-close як прямокутник (зелений = bullish, червоний = bearish)
3. Накласти EMA як криву (через `pg.PlotDataItem`)
4. Накласти Bollinger як 3 криві (upper, middle, lower)
5. Volume bars внизу окремою plot

Оцінка: ~150 рядків.

### Layout

`QMainWindow` з `QDockWidget` для 3 панелей. Користувач може
переміщувати/ховати панелі. Зберігаємо layout через `QSettings`.

Або простіше: `QSplitter` з 3 віджетами. Менш гнучко, але простіше.

Оцінка: ~50 рядків.

### Async integration

`MarketDataService.start()` запускає AsyncWorker.
`MainWindow` підключається до `klines_ready` signal.
При виборі символу → `request_klines(symbol, timeframe)`.
При отриманні результату → оновити ChartWidget + ScorePanel.

Оцінка: ~100 рядків.

### Refresh

`QTimer` кожні 60 секунд → для кожного символу викликає
`request_klines`. Показує останній timestamp в StatusBar.

Оцінка: ~50 рядків.

### Neon theme

`QPalette` з темними кольорами. Кольори:
- Background: dark gray (#1e1e1e)
- Bullish: neon green (#00ff88)
- Bearish: neon red (#ff0055)
- Neutral: neon yellow (#ffaa00)
- Text: light gray (#e0e0e0)

Оцінка: ~80 рядків.
