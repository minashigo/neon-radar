# RC2 Architecture

> Робочий документ для розвитку Neon Radar після RC1.
>
> **Статус:** План. Не реалізовано.

------------------------------------------------------------------------

# 1. Event Bus

## Мета

Відокремити бізнес-логіку від Qt та UI.

## Причина

Зараз Application частково знає про Qt Signals. У майбутньому ядро має
працювати без GUI.

## План

-   Створити внутрішній Event Bus.
-   Передавати події через нього.
-   UI лише підписується на події.

Статус: ⏳ Заплановано.

------------------------------------------------------------------------

# 2. Execution Broker

## Мета

Відокремити генерацію сигналів від виконання ордерів.

## Майбутній ланцюжок

Signal → Risk Manager → Execution Broker → Exchange

Broker повинен однаково працювати для: - Paper Trading; - Live
Trading; - майбутніх тестових середовищ.

Статус: ⏳ Заплановано.

------------------------------------------------------------------------

# 3. Portfolio Repository

## Мета

Ізолювати збереження портфеля.

Навіть якщо зараз використовується portfolio.json, доступ до нього має
виконуватись через Repository.

Статус: ⏳ Заплановано.

------------------------------------------------------------------------

# 4. Risk Manager

Перед відкриттям позиції має перевіряти:

-   максимальний ризик;
-   максимальну кількість відкритих позицій;
-   допустиме навантаження;
-   інші майбутні правила.

Статус: ⏳ Заплановано.

------------------------------------------------------------------------

# 5. Feature Store

Єдине місце для зберігання розрахованих індикаторів і ознак.

Статус: ⏳ Ідея.

------------------------------------------------------------------------

# 6. Hot / Cold Path

## Hot Path

Критичний код, що працює в реальному часі.

## Cold Path

Аналітика, звіти, статистика, бекести.

Статус: ⏳ Ідея.

------------------------------------------------------------------------

До завершення тривалого Paper Trading RC1 жоден із пунктів не
реалізовується без окремого рішення.
